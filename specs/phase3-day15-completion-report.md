## 📡 API 端点（userAnswers.js）

### 1. POST /api/user-answers/batch
**批量提交答案并自动批改**

请求：
```json
{
  "questionSetId": 123,
  "answers": [
    {
      "questionId": 1,
      "userAnswer": "C",
      "timeSpent": 120
    }
  ]
}
```

响应：
```json
{
  "success": true,
  "data": {
    "answers": [ /* 保存的答案 */ ],
    "gradingResults": [ /* 批改结果 */ ],
    "stats": {
      "total": 10,
      "correct": 8,
      "totalScore": 85,
      "accuracy": "80.0"
    }
  }
}
```

### 2. POST /api/user-answers/:id/grade
**重新批改单个答案**

用于：
- 答案修改后重新评分
- AI 批改失败后重试
- 批改标准调整后重新评分

### 3. GET /api/user-answers
**获取用户答案列表**

支持筛选：
- questionSetId - 按试卷筛选
- questionId - 按题目筛选

### 4. GET /api/user-answers/:id
**获取单个答案详情**

返回完整的答案记录。

### 5. GET /api/user-answers/:id/feedback
**获取答案的详细反馈**

返回：
- 答案记录
- 题目信息
- 批改反馈
- 得分和正确性

### 6. GET /api/user-answers/stats/summary
**获取用户答题统计摘要**

返回：
- 总题数
- 正确数
- 正确率
- 总分
- 平均分
- 总用时
- 平均用时

## 🤖 AI 批改 Prompt 设计

### Prompt 结构

1. **题目信息**
   - 题目类型
   - 题目内容
   - 标准答案
   - 答案解析

2. **学生答案**
   - 用户提交的答案
   - 未作答提示

3. **评分标准**（0-10分）
   - 10分：完全正确，逻辑清晰，表达准确
   - 8-9分：基本正确，有小瑕疵
   - 6-7分：部分正确，有明显错误
   - 4-5分：思路有一定道理，但错误较多
   - 2-3分：基本错误，仅有少量正确内容
   - 0-1分：完全错误或未作答

4. **反馈要求**
   - strengths - 答案的优点（数组）
   - weaknesses - 答案的不足（数组）
   - suggestions - 改进建议（数组）
   - encouragement - 鼓励性评语（字符串）
   - summary - 总结（字符串）

5. **输出格式**
   - 严格的 JSON 格式
   - 必需字段验证
   - 容错处理

### Prompt 特点

- ✅ 详细的评分标准
- ✅ 具体的反馈要求
- ✅ 鼓励性导向
- ✅ 结构化输出
- ✅ 温度参数优化（0.3，保证一致性）

## 🔄 批改流程

```
用户提交答案
  ↓
POST /api/user-answers/batch
  ↓
遍历每个答案
  ↓
根据题型选择批改方式
  ├─ 选择题 → 自动批改（字符串比较）
  ├─ 填空题 → 自动批改（标准化比较）
  ├─ 计算题 → 半自动批改（数值比较）
  └─ 主观题 → AI 批改
      ↓
      构建批改 Prompt
      ↓
      调用 AI API（温度 0.3）
      ↓
      解析 JSON 响应
      ↓
      验证必需字段
      ↓
      返回批改结果
  ↓
保存批改结果到数据库
  ↓
计算统计信息
  ↓
返回完整结果
```
