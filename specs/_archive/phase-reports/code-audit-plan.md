# A-Level Math App 代码审核计划

## 审核概述

**审核目标**：全面审核现有代码库，识别影响用户体验的功能问题、代码质量问题和安全隐患

**审核范围**：
- 主应用文件：`src/alevel-math-app.jsx` (4751行 - 超大单文件)
- 数据文件：`src/data/subjects.js` (超大数据文件)
- 组件：`src/components/`
- Hooks：`src/hooks/`
- 工具函数：`src/utils/`

**审核优先级**：
1. **P0 - 严重影响使用体验的功能问题**
2. **P1 - 代码架构和可维护性问题**
3. **P2 - 安全性和性能问题**
4. **P3 - 代码规范和最佳实践**

---

## P0：严重功能问题（第一优先级）

### 1.1 单文件巨石架构 - 严重影响开发和维护
**问题**：`alevel-math-app.jsx` 包含 4751 行代码，所有功能混在一个文件中
- 168 个函数/组件定义
- 55 个 React Hooks 调用
- 包含数据、UI、逻辑、样式全部混在一起

**影响**：
- 代码编辑器性能下降（语法高亮、智能提示卡顿）
- 无法快速定位问题
- 团队协作冲突频繁
- 新功能添加困难

**审核任务**：
- [ ] 分析文件结构，识别可拆分的模块
- [ ] 检查组件职责是否清晰
- [ ] 评估拆分方案的可行性

### 1.2 数据文件过大 - 影响加载性能
**问题**：`subjects.js` 包含 38090 tokens（约 150KB+）
- 所有科目数据（数学、经济、历史等）全部在一个文件
- 包含大量 YouTube 视频链接、章节详情
- 首次加载时全部解析

**影响**：
- 首屏加载时间长
- 内存占用高
- 用户只看一个科目，却加载了所有科目数据

**审核任务**：
- [ ] 检查数据加载策略
- [ ] 评估按需加载的可行性
- [ ] 检查是否有数据冗余

### 1.3 用户体验关键问题
**审核任务**：
- [ ] 检查移动端响应式设计（样式是否适配小屏幕）
- [ ] 检查 KaTeX 数学公式渲染性能（大量公式时是否卡顿）
- [ ] 检查错题本功能是否正常工作
- [ ] 检查语言切换功能是否完整
- [ ] 检查练习题生成逻辑是否有 bug
- [ ] 检查模拟考试计时器是否准确
- [ ] 检查 localStorage 存储是否有容量限制问题

### 1.4 关键交互流程测试
**审核任务**：
- [ ] 科目选择 → 章节浏览 → 练习题 → 错题本 完整流程
- [ ] 模拟考试流程（开始 → 答题 → 提交 → 查看结果）
- [ ] 视频资源链接是否有效
- [ ] 错误边界是否能正确捕获异常
