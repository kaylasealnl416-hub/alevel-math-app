# A-Level AI 教师产品规划与开发路线图

**文档版本**：v1.0
**创建日期**：2026-03-06
**产品定位**：AI驱动的A-Level全科智能教学平台

---

## 一、产品愿景与定位

### 1.1 产品愿景
打造一位"永不疲倦、因材施教"的AI教师，为A-Level学生提供全天候、个性化的学习体验，覆盖教学、练习、考试、反馈的完整学习闭环。

### 1.2 核心定位
- **不是**：传统在线课程平台 + AI助手
- **而是**：AI教师为核心 + 优质内容为支撑的智能教学系统

### 1.3 目标用户画像
- **主要用户**：15-18岁A-Level在读学生
- **次要用户**：家长（监督学习进度）、教师（辅助教学）
- **地域**：中国大陆（微信生态）+ 海外华人学生

### 1.4 核心价值主张
1. **AI主导教学**：不是辅助工具，而是真正的"教师"
2. **苏格拉底式引导**：启发思考，而非直接给答案
3. **自适应学习路径**：基于知识图谱的个性化推荐
4. **即时反馈与纠错**：实时诊断薄弱点，针对性强化
5. **游戏化激励**：成就系统、排行榜、学习勋章

## 二、竞品分析与差异化策略

### 2.1 国际竞品
| 产品 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|--------------|
| **Khanmigo** | 苏格拉底式教学、GPT-4驱动 | 泛化教育，非A-Level专精 | 专注A-Level课程体系 |
| **Squirrel AI** | 自适应学习、知识图谱 | 主要面向K12，非国际课程 | 国际课程+中国用户习惯 |
| **Integral Maths** | A-Level数学专业 | 缺乏AI互动，传统题库模式 | AI驱动的互动教学 |

### 2.2 国内竞品
| 产品 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|--------------|
| **作业帮/猿辅导** | 成熟的AI题库、拍照搜题 | 主要面向国内课程体系 | 专注A-Level国际课程 |
| **匠邦AI** | 教师工具（备课、组卷） | ToB产品，非学生端 | ToC学生端，完整学习闭环 |

### 2.3 核心差异化
1. **垂直深耕A-Level**：不做泛化教育，专注国际课程
2. **AI为第一性**：AI不是功能，而是产品核心
3. **微信生态集成**：扫码登录、小程序、社交分享
4. **视频+文本+AI三位一体**：YouTube优质视频 + 结构化知识点 + AI互动答疑

### 3.2 核心功能详细设计

#### 模块1：用户系统
**功能点**：
- 微信扫码登录（OAuth 2.0）
- 用户画像（年级、目标大学、薄弱科目）
- 学习档案（学习时长、完成题目数、知识点掌握度）
- 成就系统（勋章、等级、积分）

**技术实现**：
- 微信开放平台 OAuth 2.0
- JWT Token 认证
- PostgreSQL 用户数据存储

#### 模块2：学习系统 - 课程体系
**数据结构**：
```
Subject（科目）
  └─ Unit（单元）
      └─ Chapter（章节）
          ├─ Topic（主题/知识点）
          │   ├─ 文本内容（Markdown + KaTeX）
          │   ├─ 示例题
          │   └─ AI互动练习
          └─ Video（视频课程）
              ├─ YouTube链接
              ├─ 时间戳标注
              └─ 配套笔记
```

**功能点**：
- 结构化课程导航
- 知识点卡片（定义、公式、例题）
- 视频播放器（支持倍速、字幕、笔记）
- 学习进度追踪（已学/未学/复习）

#### 模块3：AI教师系统
**核心能力**：
1. **AI教学对话**
   - 苏格拉底式提问（不直接给答案）
   - 多轮对话理解学生困惑
   - 个性化解释（根据学生水平调整难度）

2. **AI出题引擎**
   - 基于知识图谱生成题目
   - 难度自适应（根据学生水平）
   - 题型多样化（选择、填空、计算、证明）

3. **AI批改与反馈**
   - 自动批改客观题
   - AI评分主观题（数学证明、Essay）
   - 详细错因分析
   - 针对性改进建议

4. **AI学习路径推荐**
   - 诊断测试识别薄弱点
   - 动态调整学习顺序
   - 推荐复习内容

**技术实现**：
- Claude API（Opus 4.6）作为AI教师核心
- 知识图谱（Neo4j）存储知识点关系
- 向量数据库（Pinecone）存储题库语义
- Prompt Engineering 优化教学效果

#### 模块4：练习与考试
**功能点**：
1. **日常练习**
   - 章节练习（按知识点）
   - 随机练习（混合题型）
   - 错题本（自动收集）
   - 限时挑战（游戏化）

2. **模拟考试**
   - 真题模拟（历年A-Level真题）
   - AI生成试卷（难度可调）
   - 考试计时器
   - 详细成绩报告

3. **AI点评**
   - 逐题讲解
   - 知识点关联
   - 类似题推荐

#### 模块5：数据分析与报告
**功能点**：
- 学习时长统计
- 知识点掌握度热力图
- 薄弱点诊断报告
- 进步曲线图
- 与同龄人对比

**可视化**：
- Recharts / D3.js 数据可视化
- 周报/月报自动生成

#### 模块6：社交与激励
**功能点**：
- 学习圈（分享笔记、讨论题目）
- 排行榜（周榜、月榜、总榜）
- 学习小组（组队学习、互相监督）
- 成就勋章（首次满分、连续学习7天等）

└────────────────────┬────────────────────────────────────┘
                     │
        ┌────────────┼────────────┐
        │            │            │
┌───────┴──────┐ ┌──┴──────┐ ┌──┴──────────┐
│  业务服务层   │ │ AI服务层 │ │  数据服务层  │
│ - 用户服务   │ │ Claude API│ │ PostgreSQL  │

│ - 练习服务   │ │ 向量数据库│ │ OSS存储     │
│ - 考试服务   │ │ Prompt库  │ │             │
└──────────────┘ └───────────┘ └─────────────┘

### 4.2 技术栈选型

#### 前端
- **框架**：React 18 + Vite
- **状态管理**：Zustand（轻量级）
- **路由**：React Router v6
- **UI组件**：Ant Design / shadcn/ui
- **数学渲染**：KaTeX
- **图表**：Recharts
- **视频播放**：React Player

#### 后端
- **运行时**：Node.js 20 + Bun
- **框架**：Hono（轻量高性能）
- **数据库**：PostgreSQL 16（主库）+ Redis（缓存）
- **ORM**：Drizzle ORM
- **认证**：JWT + 微信OAuth
- **文件存储**：阿里云OSS / Vercel Blob

#### AI层
- **LLM**：Claude Opus 4.6（Anthropic API）
- **知识图谱**：Neo4j（可选，初期可用关系型数据库）
- **向量数据库**：Pinecone / Supabase Vector（题库语义搜索）
- **Prompt管理**：LangChain / 自研Prompt库

#### 部署
- **前端**：Vercel（已配置）
- **后端**：Railway / Render / 阿里云
- **数据库**：Supabase（PostgreSQL托管）
- **CDN**：Cloudflare

## 六、创新点与亮点功能

### 6.1 AI教师的"人格化"
- 给AI教师设定人格（严格型、鼓励型、幽默型）
- 学生可以选择喜欢的教学风格
- AI记住学生的学习习惯和偏好

### 6.2 "学习伙伴"模式
- AI扮演学习伙伴，而非高高在上的老师
- 使用第一人称对话（"我们一起来看这道题"）
- 适当的鼓励和幽默

### 6.3 "错题变形"功能
- 学生做错一道题后，AI生成类似题目
- 直到学生真正掌握该知识点
- 避免死记硬背

### 6.4 "知识点可视化"
- 用思维导图展示知识点关系
- 学生可以看到自己的知识图谱
- 已掌握的点亮，未掌握的灰色

### 6.5 "AI学习计划师"
- 根据考试日期倒推学习计划
- 每天推送学习任务
- 动态调整计划（如果某天没完成）

### 6.6 "语音答疑"（未来）
- 学生可以语音提问
- AI语音回答（TTS）
- 更自然的交互体验

### 6.7 "AR数学实验室"（未来）
- 用AR可视化抽象数学概念
- 例如：3D几何、函数图像

## 八、风险与挑战

### 8.1 技术风险
- **AI成本**：Claude API调用成本较高
  - **应对**：缓存常见问题、限制免费用户调用次数
- **数据安全**：用户学习数据敏感
  - **应对**：数据加密、合规审计

### 8.2 内容风险
- **版权问题**：YouTube视频可能被删除
  - **应对**：建立自有视频库、多平台备份
- **题库质量**：AI生成题目可能有错误
  - **应对**：人工审核、用户反馈机制

### 8.3 市场风险
- **竞争激烈**：教育赛道巨头林立
  - **应对**：垂直深耕A-Level，建立壁垒
- **用户获取成本高**：教育产品需要口碑积累
  - **应对**：社交裂变、学校合作

## 九、成功指标（KPI）

- **AI出题准确率**：题目质量人工抽检
- **AI批改准确率**：与人工批改对比
