## 最严重的 5 个问题

### 1. 巨石文件架构 - 4751 行单文件 🔴

**问题**：`src/alevel-math-app.jsx` 包含 4,751 行代码，所有功能混在一起
- 10 个组件全部定义在同一文件
- 40+ 个 useState，3 个 useRef
- 100+ 个内联样式对象
- 2000+ 行数据定义

**影响**：
- 开发体验极差（编辑器卡顿）
- 团队协作困难（Git 冲突频繁）
- 维护成本极高（修改困难）

**修复方案**：拆分为模块化结构（8 个视图组件 + 7 个共享组件）
**预计时间**：2 周

---

### 2. 无响应式设计 - 移动端不可用 🔴

**问题**：应用完全没有响应式设计
- 无 `@media` 查询
- 固定像素值
- 移动端体验极差

**影响**：
- 50%+ 移动端用户无法使用
- SEO 排名受影响
- 用户流失严重

**修复方案**：添加响应式设计（媒体查询 + 相对单位 + 移动端适配）
**预计时间**：1 周

---

### 3. 数据文件过大 - 168KB 单文件 🔴

**问题**：`src/data/subjects.js` 包含 2,517 行代码，168KB
- 所有科目数据一次性加载
- 用户只看一个科目，却加载了全部
- 首屏加载时间 > 3 秒

**影响**：
- 首屏加载慢
- 内存占用高
- 移动网络体验差

**修复方案**：实施按需加载（动态导入 + 数据拆分）
**预计时间**：1 周

---

### 4. 代码重复严重 - 30% 重复代码 🔴

**问题**：存在 8 个主要重复模式
- API 密钥表单重复 3 次
- 选项渲染重复 3 次
- 计时器逻辑重复 2 次
- 约 1500 行重复代码

**影响**：
- 维护成本高（修改需要同步多处）
- Bug 风险高（重复代码中的 bug 会在多处出现）
- 代码膨胀

**修复方案**：提取共享组件和工具函数
**预计时间**：1 周

---

### 5. 状态管理混乱 - 40+ useState 🔴

**问题**：状态分散在多个组件
- 主应用 13 个 useState
- 整个应用 40+ 个 useState
- 无统一状态管理方案

**影响**：
- 开发困难（状态依赖不清晰）
- 性能问题（不必要的重渲染）
- 测试困难

**修复方案**：引入统一状态管理（Context + useReducer 或 Zustand）
**预计时间**：2 周

---
