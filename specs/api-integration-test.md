# API 集成测试指南

## 🎯 测试目标

验证前端可以成功从后端API获取数据，同时保持现有功能完全正常。

---

## 📋 测试步骤

### 1. 测试本地数据模式（默认）

**访问**：`http://localhost:3000`

**预期结果**：
- ✅ 应用正常显示
- ✅ 所有科目可以正常浏览
- ✅ 控制台显示：`📊 数据源模式: LOCAL`

---

### 2. 测试API数据模式

#### 步骤 2.1：访问API测试页面

**访问**：`http://localhost:3000?test=api`

**预期结果**：
- ✅ 显示 "🧪 API 测试页面"
- ✅ 显示 "数据源模式: LOCAL"（因为 VITE_USE_API=false）
- ✅ 显示5个科目卡片（经济学、历史、政治、心理学、进阶数学）

#### 步骤 2.2：测试科目详情

1. **点击任意科目卡片**（例如：经济学）
2. **预期结果**：
   - ✅ 显示科目详情
   - ✅ 显示单元列表
   - ✅ 显示章节列表
   - ✅ 控制台显示：`✅ 科目详情加载成功: 经济学`

#### 步骤 2.3：启用API模式

1. **编辑文件**：`.env.local`
2. **修改**：`VITE_USE_API=false` → `VITE_USE_API=true`
3. **重启前端**：
   ```bash
   # 停止前端（Ctrl+C）
   bun run dev
   ```
4. **再次访问**：`http://localhost:3000?test=api`
5. **预期结果**：
   - ✅ 显示 "数据源模式: API"
   - ✅ 控制台显示：`🌐 从API获取科目数据...`
   - ✅ 控制台显示：`✅ 成功获取 5 个科目`
   - ✅ 科目数据与本地模式完全一致

---

### 3. 测试降级策略

#### 步骤 3.1：停止后端服务器

```bash
# 停止后端
# （找到运行 bun run dev 的终端，按 Ctrl+C）
```

#### 步骤 3.2：刷新API测试页面

**访问**：`http://localhost:3000?test=api`（确保 VITE_USE_API=true）

**预期结果**：
- ✅ 控制台显示：`⚠️ API请求失败，降级到本地数据`
- ✅ 页面仍然正常显示科目列表
- ✅ 数据来自本地备份

---

### 4. 验证主应用不受影响

#### 步骤 4.1：访问主应用

**访问**：`http://localhost:3000`（不带 ?test=api 参数）

**预期结果**：
- ✅ 主应用正常显示
- ✅ 所有功能正常工作
- ✅ 不受API模式影响
