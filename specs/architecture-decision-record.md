### 问题发现
在开发过程中发现原始设计文档（DESIGN.md）与实际实施方案（complete-technical-plan.md）存在冲突：

## 🤔 方案对比分析

### 方案 A：纯前端方案

**架构**：
```
React + Vite
  ├── 数据：硬编码在 src/data/subjects.js
  ├── 用户数据：IndexedDB（浏览器端）
  ├── 学习进度：IndexedDB（浏览器端）
  └── AI 调用：Vercel Serverless Functions
```

**优点**：
- ✅ 简单：无需维护后端
- ✅ 免费：完全使用 Vercel 免费额度
- ✅ 快速：开发速度快

**缺点**：
- ❌ **安全性差**：API Key 必须暴露在前端代码中
- ❌ **无法同步**：学习进度仅存本地，换设备丢失
- ❌ **扩展性弱**：无法实现复杂功能（练习系统、考试系统、社交功能）
- ❌ **用户管理难**：无法实现真正的用户系统
- ❌ **数据分析难**：无法聚合用户数据

### 方案 B：前后端分离方案（当前）

**架构**：
```
前端（React + Vite）
  PS API
后端（Node.js + Hono）
  ↓ PostgreSQL
数据库（Supabase）
```

**优点**：
- ✅ **安全性强**：API Key 存储在服务器端
- ✅ **多设备同步**：学习进度云端存储
- ✅ **扩展性强**：支持所有未来功能
- ✅ **用户管理完善**：JWT 认证、权限控制
- ✅ **数据分析能力**：可聚合分析用户数据
- ✅ **已投入完成**：代码已写好，弃用浪费

**缺点**：
- ⚠️ 复杂度高：需要维护后端
- ⚠️ 成本：$0-5/月（前期免费额度内）

## 🎯 最终决策

### ✅ 决策：保留前后端分离架构

### 决策理由

#### 1. 安全性问题（关键因素）

**Claude API 安全风险**：
```javascript
// ❌ 纯前端方案 - 极度危险
const ANTHROPIC_API_KEY = "sk-ant-xxx" // 暴露在前端代码中
// 任何人打开浏览器开发者工具就能看到 API Key
// 可以无限调用，刷爆信用卡
```

**Claude API 费用**：$15/百万 tokens
- API Key 泄露 = 财务风险
- 一天可能损失数千美元

**后端方案**：
```javascript
// ✅ API Key 存储在服务器端环境变量
// 前端无法访问
// 可以实施速率限制、用户配额
```

#### 2. 产品路线图需求

**Phase 2-8 功能需求**：
- ✅ AI 对话（Phase 2）- **必须有后端**（API Key 安全）
- ✅ 练习系统（Phase 3）- 需要题库管理、防作弊
- ✅ 考试系统（Phase 4）- 需要成绩管理、时间控制
- ✅ 自适应学习（Phase 5）- 需要算法分析
- ✅ 数据分析（Phase 6）- 需要聚合用户数据
- ✅ 社交激励（Phase 7）- 需要用户交互

**纯前端方案无法实现 80% 的产品功能**

#### 3. 用户体验

**多设备同步**：
```
纯前端：
- 手机学习 → 进度保存在手机
- 电脑学习 → 进度丢失 ❌

有后端：
- 任何设备登录 → 进度云端同步 ✅
```

#### 4. 已投入成本

- 后端代码：2000 行（2 天工作量）
- 数据库设计：8 个表
- 安全系统：JWT + 速率限制 + 输入验证
- **弃用 = 浪费 100% 已完成工作**

#### 5. 成本可控

**免费额度**（前期 0 成本）：
- Vercel：100GB 带宽/月（免费）
- Railway：$5 信用/月（免费）
- Supabase：500MB 存储 + 2GB 带宽/月（免费）

**预估成本**：
- 前期（<100 用户）：$0/月
- 成长期（100-1000 用户）：$5-20/月
- 成熟期（>1000 用户）：$50-100/月

#### 6. 维护成本可控

**自动化部署**：
- Git push → 自动部署（无需手动操作）
- Railway 自动监控
- Supabase 自动备份

**实际维护时间**：
- 日常：0 小时/周（自动运行）
- 出问题：<1 小时/月（自动告警）

## 🔄 回滚策略

**混合方案**：
- 保留后端用于 AI 调用（安全）

- 维护时间 > 10 小时/月
