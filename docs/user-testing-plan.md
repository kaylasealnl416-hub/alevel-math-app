# A-Level Learning Hub - 用户测试计划

**版本**: 1.0
**测试日期**: 2026-03-12 - 2026-03-19（1 周）
**测试类型**: Alpha 测试（内部测试）

---

## 📋 测试目标

### 主要目标

1. **验证核心功能**: 确保所有功能正常工作
2. **发现 Bug**: 找出系统中的错误和问题
3. **评估用户体验**: 了解用户对界面和交互的感受
4. **收集反馈**: 获取用户对功能的意见和建议
5. **优化性能**: 发现性能瓶颈

### 成功标准

- ✅ 核心功能可用率 > 95%
- ✅ 用户满意度 > 4.0/5.0
- ✅ 发现并记录所有严重 Bug
- ✅ 收集至少 20 条有效反馈

---

## 👥 测试用户

### 招募目标

**数量**: 10-15 人
**类型**: A-Level 学生或相关人群

### 用户分组

**组 1: 数学学生**（5 人）
- 主要测试数学相关功能
- 重点：AI 教师、练习系统、考试系统

**组 2: 经济学学生**（3 人）
- 主要测试经济学相关功能
- 重点：AI 教师、学习计划

**组 3: 混合用户**（2-5 人）
- 测试多科目学习
- 重点：整体用户体验

### 招募渠道

1. **学校/培训机构**
   - 联系 A-Level 学校
   - 寻找愿意参与测试的学生

2. **社交媒体**
   - 微信群、QQ 群
   - 学习论坛

3. **朋友推荐**
   - 通过朋友介绍
   - 口碑传播

### 激励措施

**参与奖励**:
- 🎁 完成测试：获得感谢证书

## 📊 反馈收集

### 1. 每日反馈表

**问题列表**:
1. 今天使用了哪些功能？
2. 遇到了什么问题？
3. 哪些功能体验好？
4. 哪些功能需要改进？
5. 有什么建议？

**评分项**（1-5 分）:
- 整体满意度
- 界面设计
- 功能实用性
- 响应速度
- AI 教师质量

### 2. Bug 报告表

**必填信息**:
- Bug 标题
- 严重程度（严重/一般/轻微）
- 复现步骤
- 预期结果
- 实际结果
- 截图（可选）

**Bug 优先级**:
- **P0（严重）**: 系统崩溃、数据丢失、无法使用
- **P1（重要）**: 核心功能异常、影响使用
- **P2（一般）**: 次要功能问题、体验不佳
- **P3（轻微）**: 界面问题、文案错误

### 3. 最终问卷

**问题列表**:
1. 你会推荐这个平台给朋友吗？（1-10 分）
2. 最喜欢的功能是什么？
3. 最不满意的地方是什么？
4. 希望增加什么功能？
5. 对 AI 教师的评价？
6. 对题目质量的评价？
7. 对界面设计的评价？
8. 其他建议？

## 🐛 Bug 管理

### Bug 跟踪

**使用工具**: GitHub Issues

**Bug 标签**:
- `bug` - Bug 报告
- `P0` - 严重
- `P1` - 重要
- `P2` - 一般
- `P3` - 轻微
- `frontend` - 前端问题
- `backend` - 后端问题
- `ui` - 界面问题

### Bug 处理流程

1. **报告**: 用户提交 Bug
2. **确认**: 开发者确认 Bug
3. **分类**: 设置优先级和标签
4. **修复**: 开发者修复 Bug
5. **验证**: 用户验证修复
6. **关闭**: 确认修复后关闭
