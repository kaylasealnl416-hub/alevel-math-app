# Session 2026-03-18 — Upload UX overhaul, answer import, bug fixes

## Summary

Full overhaul of the question upload review experience, new "Import Official Answers" feature, and several bug fixes surfaced by code review.

## Changes

### QuestionUploadPage.jsx — complete rewrite of UI + new features

**Visual redesign (Google-style)**
- Grid background pattern, lucide-react icons (Database, Sparkles, UploadCloud)
- Dark CTA button, blue Tips card, consistent inline styles throughout
- Replaced all Tailwind layout classes with inline styles for reliability

**Bug fixes from previous session**
- Fixed `{en, zh}` React render error: `s.name?.en || s.name`, `unit.title?.en`, `ch.title?.en`
- Polling failure tolerance raised from 5 → 15 retries (prevents false upload-abort)

**Short answer textarea fix**
- When `q.type === 'short_answer'`, Answer field is now a resizable `<textarea rows={5}>` instead of a single-line input

**Import Official Answers (new feature)**
- Speed-fill bar at top of Review section
- Parses answer strings in multiple formats:
  - Concatenated: `ABCDACB`
  - Comma-separated (multi-answer): `A,B,ABD`
  - Numbered: `1A 2B 3C` or `1. A 2. B`
- Shows result: "Applied N answers · X matched AI · Y corrected"
- Official answers highlighted with green border on Answer field
- Enter key triggers Apply

**Code review fixes (4 items)**
- `importResult.applied` now uses `Math.min(answers.length, questions.length)` (was overcounting)
- Manual edit after import resets `source` from `'official'` back to `'manual'` (green border clears)
- Empty Apply click now shows red error message instead of silent no-op
- `answer.source` (UI metadata) stripped before batch save to DB
