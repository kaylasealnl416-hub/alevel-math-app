# Session 2026-03-17: Auth Fix, Question Upload, Deploy Update

## Overview

Major session focused on fixing authentication flow, getting the question upload feature working end-to-end, and deploying all accumulated changes to Vercel production.

## Changes Made

### 1. Login/Register Pages — Google OAuth Fix (Production)

**Problem**: Production site (alevel-math-app.vercel.app/login) showed "Missing required parameter client_id" error.

**Root Cause**: `AuthPage.jsx` used `useGoogleLogin` from `@react-oauth/google`, which requires `VITE_GOOGLE_CLIENT_ID`. This env var was not configured on Vercel.

**Fix**: Switched routes from `AuthPage` to standalone `LoginPage` and `RegisterPage` which don't depend on Google OAuth. Removed `GoogleOAuthProvider` wrapper from `AppRouter.jsx`.

**Files Changed**:
- `src/AppRouter.jsx` — route switch + remove GoogleOAuthProvider

### 2. Cross-Tab Authentication Fix

**Problem**: Opening app in a new tab redirected to login even when logged in, because `sessionStorage` is per-tab.

**Fix**: `AuthContext.jsx` now calls `GET /api/auth/me` with httpOnly cookie when sessionStorage has no user data (new tab scenario). Backend `/api/auth/me` endpoint updated to read auth_token from cookie in addition to Authorization header.

**Files Changed**:
- `src/contexts/AuthContext.jsx` — cookie-based auth fallback
- `backend/src/routes/auth.js` — /me endpoint reads cookie

### 3. Question Upload Pipeline — End-to-End Fix

**Problem**: Upload feature had multiple broken points preventing it from working.

**Fixes Applied** (in order of discovery):
1. **Wrong API base URL**: `import.meta.env.VITE_API_BASE_URL` was undefined → use `API_BASE` constant
2. **Auth method mismatch**: `localStorage.getItem('auth_token')` was null (token in httpOnly cookie) → use `credentials: 'include'`
3. **Toast.error not a function**: Toast.jsx is a React component, not a utility → created inline Toast helper
4. **CSRF token missing (403)**: Upload POST blocked by CSRF middleware → fetch CSRF token before upload, include X-CSRF-Token header
5. **GLM API key not configured**: Added GLM_API_KEY to backend .env.local
6. **Polling URL missing /api prefix**: `get('/questions/upload/...')` → `get('/api/questions/upload/...')`
7. **JSON parse error**: GLM returns JSON wrapped in markdown code blocks → strip ```json...``` before parsing
8. **Polling too fragile**: Single failure killed polling → allow 5 consecutive failures, increase interval to 3s
9. **Rate limit hit (429)**: Global rate limiter too strict for dev → increased dev limit to 1000 requests/15min

**Files Changed**:
- `src/components/QuestionUploadPage.jsx` — all upload/polling fixes + dynamic chapter select
- `backend/src/services/questionExtractor.js` — GLM API + JSON parsing fix
- `backend/src/index.js` — dev rate limit increase

### 4. Dynamic Chapter Selection in Upload Review

**Problem**: Chapter select dropdown had only 4 hardcoded options with fake IDs.

**Fix**: Two-level dynamic select — first choose subject (loaded from `/api/subjects`), then choose chapter (loaded from `/api/subjects/:id`, grouped by unit with `<optgroup>`).

**Files Changed**:
- `src/components/QuestionUploadPage.jsx` — dynamic subject/chapter loading
