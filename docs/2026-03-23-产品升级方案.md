# 2026-03-23 产品升级方案 — alevel-math-app

## Context

基于第三方分析文档（`网站分析：优势、劣势与发展.md`）与网站实际状态的全面对比，筛选出有价值的改进项，形成可执行的开发计划。

## 核心约束

1. **Edexcel IAL 专属** — 所有新增内容必须基于 Pearson Edexcel IAL 官方考纲，CIE/AQA/OCR 暂不考虑
2. **零破坏原则** — 已完成并使用的功能不受任何影响，所有改动向后兼容
3. **执行顺序** — 先做不影响整体的小改进 → 再做重要功能 → 最后做内容完善
4. **内容真实性** — 新增的知识点、考纲结构、AO 标准必须可在 Edexcel IAL 官网查证

## 第二部分：实施路线图

### 阶段 A — 零风险改进（不影响现有功能）

这些改动是纯增量的，不修改任何现有逻辑。

#### A1. 视频嵌入播放器增强
- **现状**: ChapterView Videos 标签页只显示链接列表
- **改动**: 增加 iframe 嵌入播放，点击视频直接在页面内播放
- **文件**: `src/components/home/ChapterView.jsx`（Videos 标签页部分）
- **风险**: 零 — 纯 UI 增强，不改数据结构

#### A2. 面包屑导航
- **现状**: 用户在 ChapterView 中不知道自己处于课程树的哪个位置
- **改动**: ChapterView / CurriculumView 顶部添加 `Mathematics > P1 > Ch3: Equations` 面包屑
- **文件**: `src/components/home/ChapterView.jsx`, `src/components/home/CurriculumView.jsx`
- **风险**: 零 — 纯新增 UI 元素

#### A3. 非数学学科补充 YouTube 视频数据
- **现状**: 经济/历史/政治/心理学章节没有 youtube 数组
- **改动**: 在 `subjects.js` 中为各章节添加 youtube 视频链接
- **前提**: 所有视频必须对应 Edexcel IAL 考纲内容
- **推荐频道**:
  - 经济: EconplusDal, Tutor2u Economics
  - 历史: History Matters, Crash Course History
  - 政治: Tutor2u Politics, TLDRNews
  - 心理: Tutor2u Psychology, Crash Course Psychology
- **文件**: `src/data/subjects.js`
- **风险**: 零 — 只增加数据字段，现有代码已支持 youtube 数组渲染

---

### 阶段 B — 重要功能增强

#### B1. 错题本云端同步
- **现状**: 前端 localStorage 存储，后端 wrongQuestions API 已存在但未对接
- **改动**: 登录状态下自动调用后端 API 同步，离线仍用 localStorage
- **文件**: `src/alevel-math-app.jsx`（errorBook state 初始化 + useEffect 同步逻辑）
- **后端**: 已有 `/api/wrong-questions` 端点，无需修改
- **安全**: 先读取云端数据合并本地，不覆盖

#### B2. AO 评分体系（Edexcel IAL Assessment Objectives）
- **现状**: 后端批改是通用 AI 打分，无 AO 分项
- **改动**:
  - 后端 System Prompt 注入各科 Edexcel IAL 的 AO 定义
  - 数学: AO1(知识运用) + AO2(问题解决) + AO3(推理论证)
  - 经济: AO1(知识) + AO2(应用) + AO3(分析) + AO4(评价)
  - 历史: AO1(知识) + AO2(因果分析) + AO3(史料评估)
  - 批改返回增加 `aoScores` 字段
- **后端文件**: `backend/src/services/promptBuilder.js`, `backend/src/services/answerGrader.js`
- **前端文件**: 考试结果页增加 AO 分项展示
- **安全**: 新增字段不影响旧数据结构

#### B3. Further Math 内容深化
- **现状**: FP1-FP2, FM1, FS1 仅有 formulas，缺少 overview/hardPoints/examTips/examples/youtube
- **改动**: 按数学模块的数据结构模板补充完整内容
- **前提**: 所有内容必须对应 Edexcel IAL Further Mathematics (9FM0) 考纲
- **文件**: `src/data/subjects.js`（further_math 部分）
- **安全**: 纯数据增加，不改组件逻辑

---

### 阶段 C — 新交互组件

#### C1. 经济学预制图表组件
- **新建文件**: `src/components/charts/EconomicsChart.jsx`
- **设计**: SVG 预制图表（供需曲线、AD/AS、PPF、市场失灵图等）
- **使用位置**: 经济学 ChapterView Learn 标签页中根据章节自动显示对应图表
- **安全**: 新增独立组件，不修改现有代码

#### C2. Desmos 数学图形嵌入
- **改动**: 在数学章节 Learn 标签页中，相关章节（二次函数、三角、微积分）嵌入 Desmos iframe
- **文件**: `src/components/home/ChapterView.jsx`
- **安全**: 条件渲染，仅在 subject=mathematics 且章节匹配时显示

#### C3. 分屏阅读器（历史/政治长文阅读）
- **新建文件**: `src/components/home/SplitPaneView.jsx`
- **设计**: CSS flexbox 实现左右分栏，左侧固定史料文本，右侧 AI 对话 + 答题区
- **使用位置**: 历史/政治科目的特定章节
- **安全**: 新增独立组件

#### C4. 心理学核心实验闪卡
- **新建文件**: `src/components/study/Flashcard.jsx`
- **设计**: 翻转卡片 UI，正面问题背面答案，可标记掌握/未掌握
- **数据**: 12 大核心实验（Edexcel IAL Psychology 9PS0 官方考纲）
- **安全**: 新增独立组件

## 第四部分：验证方式

每个阶段完成后：
1. `bun run build` — 生产构建通过
2. `bun run test` — 所有测试通过（确保不破坏现有功能）
3. 浏览器手动测试：打开各科目 → 进入各章节 → 确认原有功能正常
4. 提交 + 推送 + Vercel/Render 自动部署
5. 线上验证 https://alevel-math-app.vercel.app

## 第五部分：内容审核标准

所有新增学科内容必须满足：
1. **来源可查**: 对应 Pearson Edexcel IAL 官方 Specification（考纲文件）
2. **编码正确**: 使用 Edexcel 官方单元编码（WMA11/WEC11/WHI01 等）
3. **结构对齐**: 知识点分类与官方 Specification 的 Topic/Content 一致
4. **不含 CIE 内容**: 不引入 Cambridge 9709/9489/9990/9708 等编码或专属内容
5. **YouTube 视频**: 必须是明确标注 "Edexcel" 或 "IAL" 的教学视频
